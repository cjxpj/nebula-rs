# 变量与赋值

变量是 NR 中最基本的存储单元。本章涵盖变量的作用域规则、赋值操作符的语义差异、条件赋值、内置变量以及文本切片等核心机制。理解变量系统是正确组织数据流和控制状态的前提，与[类型系统](./types)和[表达式](./expressions)密切协作。

NR 的赋值不是简单的"文本替换"——它会根据操作符和值的内容自动推断类型（[§ 类型一览](./types)），并在不同作用域间管理变量生命周期。

## 作用域

| 类型 | 作用域 | 说明 |
|------|--------|------|
| 局部变量 | 当前上下文 | 子上下文创建时重置 |
| 全局变量 | 所有子上下文 | 子上下文创建时克隆共享 |

### 作用域选择规则

NR 根据**赋值位置**自动决定变量的作用域：

- **头部区域（第一个空行之前）**中赋值的变量为**全局变量**。这些变量在词条触发、函数调用等子上下文中可被读取和修改。
- **词条、函数、循环体内部**中首次赋值的变量为**局部变量**。局部变量仅在当前执行上下文中可见，进入子函数或子上下文时会创建独立的副本。

```
// 头部区域 → 全局变量，整个文件可见
counter:0
base_url:https://api.example.com

// ← 空行分隔

// 词条内部 → 局部变量
开始
temp:%counter%             ← 可读全局变量
counter+:[%counter%+1]   ← 可修改全局变量
local_var:临时值            ← 局部变量，外部不可见
```

### 作用域注意事项

- 全局变量在**每次词条触发时保持其值**，不会重置。这使得 `counter+:1` 可以在多次触发间累加。
- 子上下文（如 `$call$` 函数调用）中读取全局变量时，获取的是**当前快照值**；修改全局变量会影响父上下文。
- 在同一作用域中，对已存在的变量再次使用 `:` 赋值会**覆盖**其值和类型。

## 赋值操作符

| 操作符 | 示例 | 语义 |
|--------|------|------|
| `:` | `name:Alice` | **自适应赋值**：值尝试解析为 JSON 字面量（数字/布尔/null/对象/数组），不成则当字符串 |
| `::` | `greeting::Hello` | **纯文本赋值**：值始终作为字符串，不解析 JSON |
| `+:` | `score+:5` | 自增 / JSON追加 / 字符串拼接 |
| `-:` | `count-:1` | 自减 |
| `*:` | `val*:2` | 乘法 / 字符串重复 |
| `/:` | `val/:2` | 除法 |
| `%:` | `val%:3` | 取余 |

> 只有 ASCII 半角冒号 `:`（U+003A）被识别为赋值操作符。中文全角冒号 `：`（U+FF1A）**不是**操作符，在文本中直接原样输出。

### `:`（自适应赋值）vs `::`（纯文本赋值）

这是 NR 中最容易被忽视但最重要的语义差异：

| 特性 | `:`（单冒号） | `::`（双冒号） |
|------|-------------|---------------|
| 值解析 | 尝试解析为 JSON 字面量 | 始终作为字符串 |
| `1` | Int(1) | Str("1") |
| `1.5` | Float(1.5) | Str("1.5") |
| `true` | Bool(true) | Str("true") |
| `null` | Null | Str("null") |
| `{"a":1}` | JSON 对象 | Str("{'a':1}") |
| `Alice` | Str("Alice") | Str("Alice") |

```
// : 自适应赋值——数字被识别为数值类型
count:100
%TYPE@count%   → "整数"

// :: 纯文本赋值——永远是字符串
id::100
%TYPE@id%      → "字符串"

// 实际影响：数学运算
[%count%+1]   → 101   （Int 运算）
// [%id%+1]   → 报错 / 意外（字符串不能加）
```

### 何时使用 `::`

- 存储**编号、ID、电话号**等不应被当作数字处理的值（如 `id::001` 不会丢掉前导零）
- 存储可能是数字字面量的**字符串**（如文件名"123.json"）
- 需要精确控制类型以避免意外数值运算的场景

**注意事项：** 当你对 `::` 赋值的变量使用 `+:`（自增）时，由于值是字符串，`+:` 会执行**字符串拼接**而非数学加法。这可能不是你想要的行为。

## 条件赋值 `?:`

```
key:@a?:@b?:fallback
```

从左到右尝试读取 JSON 路径，第一个非空、非 null、非 false 的值被赋值。

## 内置变量

| 变量 | 说明 |
|------|------|
| `%空格%` | 空格 `" "` |
| `%换行%` | 换行 `"\n"` |
| `%时间戳%` | Unix 时间戳（秒） |
| `%毫秒时间戳%` | Unix 时间戳（毫秒） |
| `%微秒时间戳%` | Unix 时间戳（微秒） |
| `%纳秒时间戳%` | Unix 时间戳（纳秒） |
| `%时间%` | 格式化本地时间 `YYYY-MM-DD HH:MM:SS`（UTC+8） |
| `%随机数M-N%` | 随机整数 `[M, N]`（含两端） |
| `%!key%` | 布尔取反：空/0/false/null → `"1"`，否则 → `"0"` |
| `%?key%` | 可选变量：不存在返回空字符串 |
| `%++var%` | 自增：先自增再返回递增后的值（变量不存在默认 0） |
| `%--var%` | 自减：先自减再返回递减后的值（变量不存在默认 0） |
| `%参数%` | 所有参数列表（JSON 数组） |
| `%参数0%` | 函数名 / 完整触发名 |
| `%参数N%` | 第 N 个参数 |
| `%_%` | 当前对象名 / 函数名 |
| `%触发%` | 当前触发词 |
| `%行数%` | 当前行号（1-based） |
| `%B64@key%` | Base64 解码 |
| `%URL编码@key%` | URL 编码 |
| `%len@key%` / `%长度@key%` | 返回变量值的字符长度 |
| `%@var[切片]%` / `@var[切片]` | 符号截取文本（Python 风格切片，支持 `%` 快捷两种写法） |
| `%func@key%` | 获取函数指针（类型为"函数"，display 为函数名） |
| `%包名.变量%` | 跨包变量引用（如 `%mypkg.x%` 读取引入包的头部变量） |

### 内置变量的生命周期

内置变量（`%xxx%`）与用户定义的变量在生命周期上有本质区别：

- **即时求值**：每次读取内置变量（如 `%时间戳%`）时，都会**重新计算**当前值。它不存储"快照"，而是每次调用运行时函数。
- **不可赋值**：不能对内置变量使用 `:` 赋值——`%时间戳%:123` 是无效的。
- **临时性**：内置变量的值仅在当前表达式求值瞬间有效。如需多次使用同一值，应将其赋值给用户变量：`ts:%时间戳%`。
- **参数变量 `%参数N%`** 在函数体内是局部变量，函数返回后失效；在词条触发时反映的是触发参数。

```
// 推荐做法：缓存内置变量
start_time:%毫秒时间戳%
// ... 多步操作 ...
elapsed:[%毫秒时间戳%-%start_time%]   ← 两次读取产生不同值，差值有意义
```

## 符号截取文本 `%@var[切片]%` / `@var[切片]`

内联文本切片语法，对变量值按字符（`.chars()`）进行 Python 风格切片。中文、emoji 等多字节字符均算一个字符。

**两种写法等价**：

- `%@var[切片]%` —— 标准写法，需 `%` 包裹
- `@var[切片]` —— 快捷写法，无需 `%` 包裹，直接解析

**基本语法**：`@变量名[start:end:step]`

| 写法 | 含义 |
|------|------|
| `@a[1:]` | 从第 2 个字符到末尾 |
| `@a[1:3]` | 第 2 ~ 3 个字符（index 1 和 2，不含 3） |
| `@a[:3]` | 前 3 个字符 |
| `@a[-3:]` | 末尾 3 个字符 |
| `@a[1:6:2]` | index 1~5，步长 2（隔一个取一个） |
| `@a[::2]` | 从头到尾，步长 2 |
| `@a[::-1]` | 反转字符串 |

```
name:你好世界NR语言

%name%             → "你好世界NR语言"
%len@name%         → "8"

// 标准写法
%@name[1:]%        → "好世界NR语言"
%@name[1:3]%       → "好世"
%@name[:2]%        → "你好"
%@name[-3:]%       → "R语言"
%@name[::-1]%      → "言语RN界世好你"

// 快捷写法（无需 % 包裹）
@name[1:]          → "好世界NR语言"
@name[1:3]         → "好世"
@name[:2]          → "你好"
@name[-3:]         → "R语言"
@name[::-1]        → "言语RN界世好你"
```

> 快捷写法 `@var[切片]` 通过方括号内含 `:` 来与 JSON 路径导航（`@json[key]`）区分。不含 `:` 的方括号仍走 JSON 路径逻辑。
