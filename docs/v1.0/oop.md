# 面向对象编程

NR 的 OOP 系统建立在词条引擎之上的上下文隔离与状态持久化机制。本章涵盖类定义、实例变量、对象创建与方法调用、自我调用、构造函数输出以及与传统 OOP 的对比。

<dl>
  <dt>类</dt>
  <dd>一组绑定到特定命名空间的方法集合，使用 <code class="nr-sig">[函数:类名]</code> 语法定义。</dd>
  <dt>实例变量</dt>
  <dd>以 <code class="nr-sig">.字段</code> 前缀命名的变量，在对象方法调用间自动持久化。</dd>
  <dt>对象</dt>
  <dd>通过 <code class="nr-sig">$new ClassName$</code> 创建，每个实例维护独立的变量作用域。</dd>
</dl>

## 类定义

使用 `[函数:类名]`、`[f:类名]` 或 `[F:类名]` 语法：

```
[函数:Counter]初始化
$打印 初始化$
.count:0

[f:Counter]add num     ← [f:类名] 是 [函数:类名] 的简写
.count+:%num%

[函数:Counter]get
当前计数：%.count%
```

- 内部触发词为 `类名.方法名`
- 构造函数查找顺序：`类名.new` → `类名.初始化`

### 类生命周期

一个 NR 对象的完整生命周期如下：

1. **创建**：通过 `$new ClassName args$` 创建对象实例，引擎查找并执行构造函数
2. **初始化**：构造函数中设置初始实例变量（`.field:value`），构造函数返回类名字符串给调用者
3. **使用**：通过 `$对象名.方法 参数$` 调用方法；每次方法调用前从存储中加载实例变量，执行后自动写回
4. **消亡**：当对象变量被覆盖或超出作用域时，实例数据随之释放

**存储机制**：实例变量存储在引擎内部的键值存储中，键的格式为 `对象名.字段名`。这意味着不同对象的 `.count` 完全独立。

## 实例变量 `.字段`

以 `.` 开头的变量是**实例变量**，在同一个对象的多次方法调用之间持久化：

```
[f:Counter]add num
.count+:%num%           ← .count 跨调用保持
```

- 方法调用前从主上下文加载（`对象名.字段`），执行后自动写回
- 不同对象的实例变量相互独立

## 创建对象与调用方法

```
obj:$new Counter$
obj:$new Counter 参数$

$obj.add 5$             ← 调用方法
$obj.get$               ← 无参调用
```

方法返回值通过 `$...$` 替换到调用处。

## 自我调用 `$.method$`

在类方法内部，使用 `$.方法名` 调用同一对象的其他方法：

```
[函数:Counter]get
$.add 1$                ← 等价于 $Counter.add 1$
当前计数：%.count%
```

- `$.method$` 通过 `self` 变量自动解析为 `ClassName.method`
- 只能在类方法内部使用（`self` 变量不为空时生效）
- 支持传参：`$.method arg1 arg2$`

## 构造函数与输出

```
[函数:Counter]初始化
.count:0
a                       ← 裸文本直接打印到终端
$打印 初始化$            ← $打印$ 输出到终端
```

- 构造函数中的裸文本直接打印到终端，不走管道输出
- `$打印$` 和 `$打印返回$` 同样输出到终端
- 构造函数返回 `类名` 字符串（可赋给变量）

## NR OOP vs 传统 OOP 对比

| 概念 | NR OOP | 传统 OOP（Java/Python） |
| --- | --- | --- |
| 类定义 | `[函数:类名]方法名` 分散定义 | `class { }` 集中定义 |
| 继承 | 不支持类继承 | `extends` / 接口实现 |
| 实例变量 | `.字段`，引擎自动持久化 | `this.field`，内存中维护 |
| 构造函数 | `类名.new` 或 `类名.初始化` | `constructor()` / `__init__()` |
| 方法调用 | `$对象.方法 args$` | `obj.method(args)` |
| 自我调用 | `$.method$` 语法糖 | `this.method()` |
| 访问控制 | 无 public/private，全公开 | public/protected/private |
| 多态 | 不支持 | 虚函数/接口 |
| 适用场景 | 对话状态、简单实体、计数器 | 通用软件工程 |

> **注意事项**
> - **无继承**：NR 不支持类继承。如需共享行为，使用[普通函数](./functions)或[模块引入](./modules)替代
> - **实例变量命名**：`.字段` 命名空间与普通变量独立，`.count` 和 `count` 是两个不同的变量
> - **构造函数返回值**：构造函数返回的是**类名字符串**（如 `"Counter"`），而非对象引用。真正的"对象标识"是赋值的变量名
> - **方法中修改实例变量**：在类方法中修改 `.字段` 后，引擎**在执行完毕后自动写回**——不需要显式的 "save" 操作
> 关于对象创建 `$new$` 的更多用法，参见[对象创建](./object)
