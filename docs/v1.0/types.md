# 类型系统

NR 的类型系统是**动态、弱类型**的——变量无需声明类型，运行时自动确定。值在大多数场景下会自动在文本和数字之间转换，`==` 运算符做宽松比较。本章涵盖 NR 中的原始数据类型（没有对象/数组，只有文本、数字和布尔值），以及类型检测、判断和转换操作。

NR 的 `Value` 不是简单的"字符串或数字"二分法——它是一个完整的枚举类型，内部由 Rust 的 `enum` 实现，每个变体携带不同的数据表示。这意味着两个值即使"看起来一样"，也可能属于不同类型，进而影响相等性比较和运算行为。

## 类型一览

NR 变量存储的是**带类型的值**（`Value`），而非纯字符串。类型在赋值时自动推断。

| 类型 | 示例 | 说明 |
|------|------|------|
| `整数` | `1`, `-5`, `100` | 64 位有符号整数 |
| `浮点` | `1.5`, `-0.5`, `3.14` | 64 位浮点数 |
| `字符串` | `"hello"`, `abc` | 字符串 |
| `布尔` | `true`, `false` | 布尔值 |
| `空` | `null` | 空值 |
| `对象` | — | 对象引用（`$new$` 返回） |
| `函数` | — | 函数指针（`%func@name%` 返回） |

- `count:5` → Int，`name:Alice` → Str
- 但 `[...]` 数学表达式运算结果保留数值类型
- `$new$` 返回值为对象类型
- `%func@key%` 返回值为函数类型

## 各类型详解

### 整数（Int）

64 位有符号整数（`i64`），范围约为 `-9.2×10¹⁸` 到 `9.2×10¹⁸`。

```
a:42
b:-100
c:0

%TYPE@a%   → "整数"
%TYPE@b%   → "整数"
%TYPE@c%   → "整数"
```

整数字面量不能包含小数点、前导零（会被当作十进制解析）或科学计数法。

### 浮点（Float）

64 位浮点数（`f64`），符合 IEEE 754 标准。

```
pi:3.14
neg:-0.5
big:1.5e10

%TYPE@pi%    → "浮点"
%TYPE@neg%   → "浮点"
%TYPE@big%   → "浮点"
```

浮点数和整数在比较（`==`）时**视为不同**：`1 == 1.0` 在严格比较下为假。浮点运算可能产生精度误差，这是 IEEE 754 浮点数的固有特性。

### 字符串（Str）

UTF-8 编码的字符串。不带特殊前缀的纯文本字面量自动判定为字符串。

```
name:Alice
greeting:Hello World
empty:""              ← 显式空字符串

%TYPE@name%       → "字符串"
%TYPE@greeting%   → "字符串"
```

注意：`greeting:Hello World` 中空格不需要转义（赋值操作符 `:` 之后直到行尾均为值部分）。但在 `$...$` 参数传递中空格会分割参数，参见 [词法结构 § 转义规则](./lexical)。

### 布尔（Bool）

只有 `true` 和 `false` 两个字面量。

```
flag:true
done:false

%TYPE@flag%   → "布尔"
%TYPE@done%   → "布尔"
```

布尔值可直接用于条件表达式（[控制流](./control-flow)），也可参与逻辑运算（[表达式 § 逻辑运算符](./expressions)）。

### 空值（Null）

表示"无值"或"未设置"。仅有一种字面量：`null`。

```
data:null

%TYPE@data%   → "空"
```

`null` 与空字符串 `""` 不同：`null` 是一个独立的类型标记，而 `""` 是值为空的字符串。在条件判断中，`null` 被视为假值。

### 对象（Object）

由 `$new$` 创建，内部持有一个 JSON 对象引用。对象类型值不可直接输出为文本，需通过 `%TYPE@` 查询或使用 JSON 操作符访问其字段。

```
obj:$new$ {"key":"val"}

%TYPE@obj%   → "对象"
```

`$new$` 不是赋值操作符——它是创建新对象的内置函数，参见 [词条系统](./entries)。

### 函数（Func）

由 `%func@name%` 获取，存储的是函数指针/引用，而非函数体文本。

```
ptr:%func@greet%

%TYPE@ptr%   → "函数"
```

函数类型值主要用途：作为回调传入其他函数，或在需要延迟调用时暂存。

## 类型特性对照

| 类型 | 可修改（`+:` 等） | 宽松相等 `==` | 严格相等 `===` | 数值排序 `>` `<` |
|------|-------------------|--------------|---------------|------------------|
| Int | ✅ 算术运算 | 字符串化后比较 | 值和类型均相同 | ✅ 按数值 |
| Float | ✅ 算术运算 | 字符串化后比较 | 值和类型均相同 | ✅ 按数值 |
| Str | ✅ 串接 | 内容比较（含类型转换） | 值和类型均相同 | ❌ 无意义 |
| Bool | ❌ | 字符串化后比较 | 值和类型均相同 | ❌ |
| Null | ❌ | — | — | ❌ |
| Object | — | 引用相等 | 引用相等 | ❌ |
| Func | ❌ | 引用相等 | 引用相等 | ❌ |

## 类型强制转换与兼容性

### 算术上下文中的类型转换

在 `[...]` 数学表达式中，操作数的类型决定了运算结果的类型：

```
// 整数运算 → 整数
a:[1+2]          → Int(3)

// 浮点数参与 → 浮点
b:[1+2.0]        → Float(3.0)
c:[3.14*2]       → Float(6.28)

// 字符串自动转为数值参与运算
d:["3"+2]        → Int(5)  或报错
```

- 整数 + 整数 → 整数
- 任一操作数为浮点 → 结果提升为浮点
- 字符串操作数：尝试按数值解析，失败则报错

### 比较上下文中的类型转换

`==`（宽松相等）会将两边值转为字符串后比较：

```
1 == "1"       → true（都转为 "1"）
1 == 1.0       → true（都转为 "1"，但用 === 则为 false）
true == "true" → true
null == ""     → false（"null" ≠ ""）
```

`===`（严格相等）要求**类型和值都相同**：

```
1 === 1       → true
1 === 1.0     → false（Int vs Float）
"1" === 1     → false（Str vs Int）
true === true → true
```

**注意事项：**

- 条件判断中（如 `如果:`），**推荐使用严格比较 `===`** 以避免意外的类型转换带来的误判。
- 数值比较（`>`、`<`、`>=`、`<=`）要求两边均为数值类型（Int 或 Float），字符串会被尝试解析为数值。
- `0`、`""`、`null`、`false` 在布尔上下文中均为"假"，但它们彼此之间**不相等**（除了 `0 == false` 会因为字符串化为 `"0" == "false"` 而返回假）。

## 类型查询 `%TYPE@var%`

```
count:5
name:Alice

%TYPE@count%   → "整数"
%TYPE@name%    → "字符串"
```

未设置变量返回空字符串。
