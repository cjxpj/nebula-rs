# 表达式与运算符

表达式是 NR 中执行计算和逻辑判断的核心机制。本章涵盖数学表达式 `[...]`、比较运算符、逻辑运算符及其优先级规则。理解运算符的优先级、结合性和类型转换行为，对于编写正确的 [条件判断](./control-flow) 和[变量赋值](./variables)至关重要。

NR 的运算符体系分为三个层级：**数学运算符**（在 `[...]` 内使用）、**比较运算符**和**逻辑运算符**。它们在表达式中按固定的优先级顺序求值。

## 数学表达式 `[...]`

```
result:[1+2*3]              → "7"
score:[%base%*2+10]
```

### 运算符优先级总表（从高到低）

| 优先级 | 类别 | 运算符 | 结合性 | 说明 |
|--------|------|--------|--------|------|
| 1（最高） | 分组 | `()` | — | 括号分组 |
| 2 | 一元 | `-` | 右→左 | 一元负号 |
| 3 | 幂 | `^` | 右→左 | 幂运算（`2^3^2 = 2^9 = 512`） |
| 4 | 乘除取余 | `*` `/` `%` | 左→右 | 乘、除、取余 |
| 5 | 加减 | `+` `-` | 左→右 | 加、减 |
| 6 | 位移 | `<<` `>>` | 左→右 | 左移、右移 |
| 7 | 比较 | `>=` `<=` `>` `<` `==` `!=` `===` `!==` `~=` `in` | — | 比较运算（非数学表达式） |
| 8 | 逻辑 AND | `&&` `&` | 左→右（短路） | 逻辑与 |
| 9（最低） | 逻辑 OR | `\|\|` `\|` | 左→右（短路） | 逻辑或 |

**结合性说明**：

- **左结合**（左→右）：`a / b / c` 解释为 `(a / b) / c`
- **右结合**（右→左）：`- -x` 解释为 `-(-x)`；`2^3^2` 解释为 `2^(3^2)`

注意：比较和逻辑运算符不在 `[...]` 数学表达式中使用——它们出现在条件语句（`如果:`、`循环>`）和 `$if$` 等结构中。数学表达式 `[...]` 内部仅使用优先级 1-6 的运算符。

- 整数运算结果仍为整数，浮点数参与则为浮点
- 位运算和幂运算强制转整数

### 混合类型运算的边界情况

当数学表达式中混合不同类型的操作数时，结果遵循以下规则：

```
// 类型提升示例
[1+2.0]        → 3.0   （Int + Float → Float）
[3/2]          → 1     （Int 除法，截断）
[3/2.0]        → 1.5   （Float 除法）

// 边界情况
[1/0]          → 报错（整数除零）
[1.0/0.0]      → inf   （浮点除零，返回无穷大）
[-2147483648]    → 溢出 (i64 边界)
[0.1+0.2]      → 0.30000000000000004  （IEEE 754 精度问题）
```

**注意事项：**

- 整数除法 `a / b` 结果为整数（向零截断），不是四舍五入。
- 幂运算 `a ^ b` 要求 b 为非负整数（包括 0）。
- 位移运算 `a << b` 中 b 必须是非负整数，且结果类型为整数。

## 比较运算符

| 操作符 | 含义 |
|--------|------|
| `==` | 字符串相等（宽松，自动类型转换） |
| `!=` | 字符串不等（宽松） |
| `===` | 严格相等（比较值和类型） |
| `!==` | 严格不等（比较值和类型） |
| `>=` `<=` `>` `<` | 数值比较 |
| `~=` | 正则匹配 |
| `in` | 包含判断 |

### 严格比较 `===` / `!==`

- `===` 要求值和**类型**都相同：`1 === 1` 为真，`1 === 1.0`（Int vs Float）为假
- `!==` 取反：类型不同或值不同时为真
- 普通 `==` / `!=` 做字符串比较，不区分类型

```
如果:%a%===1       ← 严格检查整数 1
整数 1
如果尾

如果:%a%===1.5     ← 严格检查浮点数 1.5
浮点数 1.5
如果尾
```

### 比较运算边界情况

```
// 数值比较中的类型问题
[1>0.5]          → 在条件中为真（自动类型提升比较数值）
1 == 1.0            → 条件判断中为真（宽松相等，都转为 "1"）
1 === 1.0           → 条件判断中为假（严格相等，Int vs Float）
```

- 数值比较（`>` `<` `>=` `<=`）会自动尝试将双方转为数值再比较。一方无法转数值时报错。
- `==` 宽松相等适合快速判断，但**在需要区分 Int/Float 时请用 `===`**。
- `~=`（正则匹配）右侧必须是正则模式；`in` 左侧检查是否包含于右侧（字符串子串或数组元素）。

## 逻辑运算符

| 操作符 | 含义 |
|--------|------|
| `&&` / `&` | 逻辑 AND（短路） |
| `\|\|` / `\|` | 逻辑 OR（短路） |

单操作数时做真值判断：非空、非 `"0"`、非 `"false"`、非 `"null"` 为真。
